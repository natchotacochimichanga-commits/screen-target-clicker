Features

Built for real automation sessions

Not just image matching — timing, zones, rules, and multiple instances working side by side.

Window targeting

Pick any visible window. Each app instance claims its own target so two copies never fight over the same window.

Image & rule matching

Add target images for instant clicks, or conditional rules where primary and subsection images must both match.

Scan zones

Crop where to search — a global zone for targets and separate purple zones per rule.

Live scan HUD

A compact preview pins to your target window with match markers, cooldown timer, and loop stats.

Timing controls

Global click cooldown, burst click limits, long pauses, and wait tracking so automation stays controlled.

Configs & themes

Setups auto-save while you work and reload on launch. Atlas theme, eight color presets, custom fonts, and exportable JSON configs.

Multi-instance

Run several copies at once — each with its own window, hotkey, and saved config file.

Hotkey & test mode

Toggle scanning with a global hotkey (F6 by default) and preview detection before you start.

How it works

Up and running in minutes

  1. 1

    Download & extract

    Grab the portable zip from GitHub Releases, extract it anywhere, and run ScreenTargetClicker.exe.

  2. 2

    Choose a window & targets

    Select the app to watch, add screenshot targets, and optionally draw scan zones or conditional rules.

  3. 3

    Test, then start

    Use Test Detection to verify matches, tune confidence in Settings, then hit START or your hotkey.

  4. 4

    Close when done

    Your config saves automatically. Next launch picks up where you left off.

Failsafe: move the mouse to the top-left corner of the screen to abort PyAutoGUI actions.
